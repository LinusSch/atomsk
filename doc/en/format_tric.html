Format: TRIC

Name: (Partial) input file for TRIC, Transmission of Ions through Crystals

Extension: .TRC (the file must be named INP.TRC for use with TRIC)

Specification: not available, as far as I'm aware

Visualization programs: none

Restrictions

Output only, Atomsk cannot read this format.

Currently writes zeros as placeholders for amplitudes of thermal vibrations. This is not good, as TRIC WILL HANG with no error message when run with such input.

Orthogonal cells only, TRIC supports nothing else. This can be achieved with the Atomsk option -orthogonal-cell.

There is no automatic re-sorting of the list of atoms. This can be significant as one would typically want an atom at position [000] to use as reference for positioning the surface, and therefore wants to know its place in the list. Sorting can be achieved with the Atomsk option -sort, see example below.

Fairly untested, and the format is only defined by what the TRIC code will accept as input, so bugs are expected. Report them to Linus Schönström at physics.uu.se, by email (substitute o for ö in the email address).

Examples

  • atomsk my_system.xsf tric

    This will read the file my_system.xsf, and convert it to the file my_system.TRC.

  • atomsk my_system.cif -sort z up -sort y up -sort x up sorted.trc

    This will read the file my_system.cif, sort the atom list so that those with the lowest-valued coordinates appear at the top, and write it to the file sorted.TRC.

    Note: in one test this did not quite produce the desired result, y positions appeared to have been sorted before z positions, although x positions appeared correctly sorted last.
